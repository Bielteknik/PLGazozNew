## 1. Genel Sistem Analizi

### Proje: Palandöken GAZOZ — İçecek Dolum Makinesi Kontrol Sistemi

**Mimari:**
- Frontend: React 19 + TypeScript + Vite 6 + TailwindCSS + motion + Recharts
- Backend: Python + FastAPI + python-socketio + SQLite
- Donanım: Raspberry Pi 5 (GPIO) + 2× Arduino Nano (Serial/UART)
- İletişim: Socket.IO (FE↔BE) / Serial (BE↔Nano) / GPIO (sensor polling)

**Donanım Birimleri:**
- GatesNano (NANO_1): Step motor kapılar (NEMA 17) + lazer sensörler
- ValvesNano (NANO_2): 9 adet solenoid valf
- Raspberry Pi 5: Ana kontrolör, GPIO üzerinden lazer okuma

**Sistem Modları:** BASLATMA, BEKLEMEDE, OTOMATİK, MANUEL, YIKAMA, ARIZA, TAHLIYE

## 2. Production Senaryosu Analizi

### Kullanıcının Tanımladığı Döngü:

1. Tüm kilitler kapalı → giriş kapısı açılır
2. İlk şişede çıkış kapısı açıksa kapatılır (güvenlik)
3. Reçete adedince (9 şişe) giriş lazeri sayar
4. Giriş kapısı kapanır
5. ~500ms titreşim/yerleşme beklemesi
6. Valfler açılır (3000ms) → şerbet dolumu
7. Valfler kapanır → 500ms damlatma beklemesi
8. Çıkış kapısı açılır → çıkış lazeri sayar
9. Giren == Çıkan → çıkış kapanır, yeni döngü
10. Giren != Çıkan → onay prompt'u (EVET/HAYIR)

### Yapılan Değişiklikler:

- `production_manager.py` tamamen yeniden yazıldı:
  - Sıralama: settle → fill → drip → evacuate
  - `first_bottle` güvenlik kontrolü (ilk şişede çıkış kapısını kapat)
  - `dripWaitTimeMs` kullanımı eklendi
  - `COUNT_MISMATCH` prompt mekanizması
  - Cycle passport kaydı (`db.add_cycle`)
- `ANSWER_PROMPT` handler'ı backend'e eklendi
- `COUNT_MISMATCH` overlay Dashboard'a eklendi
- `promptData` alanı TypeScript tipine eklendi
